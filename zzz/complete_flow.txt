# Deforestation Detection in the Amazon: Code Overview and Flow


# 1. Overview of the Flow
The project uses PyTorch and PyTorch Lightning to train an attention-based neural network 
   (specifically, UTAE - U-Net with Temporal Attention Encoder) to identify deforestation end-to-end from 
   sequences of satellite images. The framework is designed to handle temporal data and predict deforestation masks over time.


# About the data set 
  why SAR (Synthetic Aperture Radar):Earth observation using radar
  -Uses microwave signals
  -it can work in both day & night
  -black & white (grayscale)


# Execution Flow:
1. Initialization: `run_via_parser.py` is the entry point. It reads hyperparameters 
    via command-line arguments (e.g., dataset path, batch size, learning rate, network architecture).

2. Data Loading: The dataset (BraDD-S1TS) is loaded and pre-processed by the data modules.
    It organizes the SAR image time series and the corresponding ground-truth deforestation masks.
    
3. Model Construction:The `Experiment` class (a PyTorch Lightning Module) is instantiated, 
    bringing together the neural network (`Network`), loss function, metric calculators (`SegmentationScores`),
    and the `ForwardFunction`.

4. Training Loop: The PyTorch Lightning `Trainer` fits the model. During each step, batches of time-series
    images are passed to the model.

5. Prediction & Optimization: The model predicts deforestation masks. The `ForwardFunction` translates
    these predictions into a format suitable for the loss function (e.g., Change Detection between temporal steps). 
    The optimizer then updates the model weights.

6.Testing & Logging: The best model is evaluated on a test set, and metrics like IoU (Intersection over Union)
    are logged using WandB or TensorBoard.



#. Steps to Detect Deforestation
1. Input Data Preparation: Sequences of Sentinel-1 SAR (Synthetic Aperture Radar) images are gathered along with the timestamps (days) they were taken.

2. Temporal Modeling: The input sequences are fed into the UTAE model. Because satellite images can be taken at irregular intervals, 
    the model takes both the images and their corresponding acquisition days.
    The temporal attention mechanism helps the model focus on relevant changes over time,
    ignoring temporary artifacts like weather anomalies.

3. Change Detection: The `ForwardFunction` is set up to perform Change Detection. It doesn't just look at a single image; 
    it compares predictions across the timeline to detect *when* a pixel changed from "Forest" to "Deforested" (i.e., a change target of 0 to 1).

4. Mask Generation: The network outputs spatial logits representing the probability of a pixel being deforested at a given target timeframe.

5. Evaluation: The predicted masks are compared with the actual target masks to compute evaluation scores (e.g., CrossEntropy for loss and IoU for performance). 


#. Which File Does What

- run_via_parser.py: The main executable script. It handles argument parsing, sets up the loggers (WandB, TensorBoard, CSV),
      initializes the PyTorch Lightning Trainer, handles checkpoints, and starts the training and testing processes.
  
- source/dataset.py & source/data_module.py: Responsible for loading the BraDD-S1TS dataset from disk, 
     applying normalization (like Z-Score), handling temporal dropouts, and batching the data for the DataLoader.

- source/network.py: Defines the `Network` module which wraps the base architecture (like `UTAE`). 
    It contains the logic (`_segment_wise`, `_begin_to_end`) for how the time-series images and 
    acquisition days are batched and fed into the underlying network based on the target days.

- source/experiment.py: The core PyTorch Lightning module (`pl.LightningModule`). It glues the network, optimizer (AdamW),
     learning rate scheduler, loss, and scoring functions together. It defines what happens during `training_step`, `validation_step`, and `test_step`.

- source/forward_function.py: Contains the logic to compute predictions from the network and format them for loss and scoring. 
    Notably, it contains the `_change_detection` logic which determines if a pixel has changed state between consecutive 
    timesteps (identifying new deforestation).

- source/loss_function.py & source/loss/: Contains the loss criteria used to train the network (e.g., CrossEntropy, Dice Loss, Focal Loss).

- source/score.py: Evaluates the network's predictions against the targets using metrics like Intersection over Union (IoU), 
    which is the primary metric for this segmentation task.

- source/utae/: Contains the implementation of the U-Net with Temporal Attention Encoder (UTAE),
     which is the core neural network architecture used to process the spatial-temporal satellite data.
