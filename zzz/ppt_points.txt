# Base Paper Implementation: Deforestation in Amazon
# Presentation Points Outline (Simplified Version with Visuals & Formulas)

## 1. Introduction
* Topic: Automatically finding deforestation in the Amazon using AI.
* Why it matters: The Amazon is very important for our climate, and we need to track if trees are being cut down.
* How we do it: Using satellite images and Artificial Intelligence to watch the forest over time.
* **[Suggested Image: A side-by-side picture of a healthy Amazon forest and a recently deforested area.]**

## 2. Problem Statement
* Normal cameras on satellites can't see through the thick clouds over the Amazon.
* Looking at just one picture isn't enough; we need to know *when* the change happened.
* We need a strong computer system that can look at many pictures taken at different times to find new deforestation.
* **[Suggested Image: A satellite photo completely covered in white clouds next to a clear radar (SAR) image of the exact same spot.]**

## 3. Objectives
* Build an AI model that can understand a series of images over time.
* Use special radar images that can see through clouds.
* Find the exact time a forest changes into a deforested area.
* Make the code easy to use and run for future research.

## 4. Dataset
* Data Name: BraDD-S1TS (Brazilian Deforestation Data).
* What's inside: A series of radar images of the same place over time, plus maps that show where trees were actually cut down.
* Preparing the data: We clean the data and teach the AI how to handle it if some images are missing.

## 5. SAR Basics (Radar Images)
* What is SAR?: It's a special type of radar on satellites.
* Seeing through clouds: It uses signals that go right through rain, fog, and clouds.
* Works day and night: Since it makes its own signal, it doesn't need sunlight to take a picture.
* What it looks like: The images are black and white and show physical changes on the ground (like missing trees).
* **[Suggested Image: A diagram showing a satellite sending a radar beam down to the earth and bouncing back (scattering) differently off tall trees versus flat, cut land.]**

## 6. Architecture (The AI Model)
* Model Name: UTAE (A smart AI designed for images and time).
* How it works: It looks at the shape of the land in the images and also pays attention to *when* the image was taken.
* The "Attention" part: It learns to ignore temporary things (like a wet day) and focus on real, permanent changes (like cut trees).
* The result: It tells us the chance that a certain spot has been deforested.
* **[Suggested Image: A block diagram of the UTAE model from the base paper, showing the "Encoder", "Temporal Attention", and "Decoder" blocks.]**

## 7. Pipeline (How the Code Runs)
* 1. Start (`run_via_parser.py`): The code reads our settings (like how fast the AI should learn).
* 2. Load Data (`source/dataset.py` & `source/data_module.py`): It groups the images so the AI can read them easily.
* 3. Build Model (`source/experiment.py` & `source/network.py`): It puts together the AI, the scoring system, and the learning rules.
* 4. Find Changes (`source/forward_function.py`): It compares old and new pictures to spot when the trees disappeared.
* 5. Learn (`source/loss_function.py` & PyTorch Lightning Trainer): The AI checks its mistakes and gets better over time.
* **[Suggested Image: A simple flowchart showing: Input Images -> UTAE Model -> Change Detection -> Final Deforestation Map.]**

## 8. Implementation (The Tools We Used)
* Main Tools: PyTorch and PyTorch Lightning (popular AI building tools).
* Measuring Success: We use a score called "IoU" (Intersection over Union), which measures how perfectly our AI's map matches the real map.
  * **Formula (IoU):** `IoU = Area of Overlap / Area of Union`
  * *(In simple terms: Correct Guesses / (Correct Guesses + Missed Trees + False Alarms))*
* Loss Function: The math used to punish the AI when it guesses wrong (usually Cross-Entropy).
* Saving Progress: The code automatically saves the best version of the AI while it learns.
* **[Suggested Image: A visual diagram of IoU showing two overlapping circles (Prediction vs Ground Truth), highlighting the overlapping center section.]**

## 9. Results
* The test: We check how well the AI works on images it has never seen before.
* Finding the change: The AI is very good at telling the difference between a natural change and trees being cut down.
* Final check: The system automatically picks the best AI model to give the final scores.
* **[Suggested Image: Visual results showing 3 squares: The original SAR image, the Ground Truth mask (where trees were actually cut), and the Model Prediction mask (what the AI guessed).]**

## 10. Limitations (Challenges)
* No Colors: Because radar is black and white, it's sometimes harder to tell different types of plants apart.
* Complex Time: Satellites don't take pictures every single day, so the math gets tricky when images are spaced out randomly.
* Needs human help: The AI needs humans to show it correct examples first, which takes a lot of time to create.

## 11. Your Novelty (What makes this special)
* Finding the *moment* of change: Instead of just saying "this is a forest", the AI finds the exact time the forest was lost.
* Smart use of Radar: We successfully applied a very advanced AI model (UTAE) to black-and-white radar data.
* Clean Code: We built the whole complex system in a very neat and organized way so anyone can understand and use it.

## 12. Conclusion
* We built a strong, automatic system to track deforestation in the Amazon.
* By using radar images, bad weather and clouds don't stop us anymore.
* The code is organized well, making it easy for others to improve it in the future.
